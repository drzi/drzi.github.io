<p><a href="https://travis-ci.org/pages-themes/minimal"><img src="https://camo.githubusercontent.com/b259c8a34bdb0b671d67abf4f9c4efd184436b04/68747470733a2f2f7472617669732d63692e6f72672f70616765732d7468656d65732f6d696e696d616c2e7376673f6272616e63683d6d6173746572" alt="Build Status" data-canonical-src="https://travis-ci.org/pages-themes/minimal.svg?branch=master" style="max-width:100%;"></a> <a href="https://badge.fury.io/rb/jekyll-theme-minimal"><img src="https://camo.githubusercontent.com/ecf03273f8ed04450495a85739744c970559c30f/68747470733a2f2f62616467652e667572792e696f2f72622f6a656b796c6c2d7468656d652d6d696e696d616c2e737667" alt="Gem Version" data-canonical-src="https://badge.fury.io/rb/jekyll-theme-minimal.svg" style="max-width:100%;"></a></p>
<p><em>Minimal is a Jekyll theme for GitHub Pages. You can <a href="http://pages-themes.github.io/minimal">preview the theme to see what it looks like</a>, or even <a href="#usage">use it today</a>.</em></p>
<p><a href="/pages-themes/minimal/blob/master/thumbnail.png" target="_blank"><img src="/pages-themes/minimal/raw/master/thumbnail.png" alt="Thumbnail of minimal" style="max-width:100%;"></a></p>
<h2><a id="user-content-usage" class="anchor" href="#usage" aria-hidden="true"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Usage</h2>
<p>To use the Minimal theme:</p>
<ol>
<li>
<p>Add the following to your site's <code>_config.yml</code>:</p>
<div class="highlight highlight-source-yaml"><pre><span class="pl-ent">theme</span>: <span class="pl-s">jekyll-theme-minimal</span></pre></div>
</li>
<li>
<p>Optionally, if you'd like to preview your site on your computer, add the following to your site's <code>Gemfile</code>:</p>
<div class="highlight highlight-source-ruby"><pre><span class="pl-k">gem</span> <span class="pl-s"><span class="pl-pds">"</span>github-pages<span class="pl-pds">"</span></span>, <span class="pl-c1">group:</span> <span class="pl-c1">:jekyll_plugins</span></pre></div>
</li>
</ol>
<h2><a id="user-content-customizing" class="anchor" href="#customizing" aria-hidden="true"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Customizing</h2>
<h3><a id="user-content-configuration-variables" class="anchor" href="#configuration-variables" aria-hidden="true"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Configuration variables</h3>
<p>Minimal will respect the following variables, if set in your site's <code>_config.yml</code>:</p>
<div class="highlight highlight-source-yaml"><pre><span class="pl-ent">title</span>: <span class="pl-s">[The title of your site]</span>
<span class="pl-ent">description</span>: <span class="pl-s">[A short description of your site's purpose]</span></pre></div>
<p>Additionally, you may choose to set the following optional variables:</p>
<div class="highlight highlight-source-yaml"><pre><span class="pl-ent">show_downloads</span>: <span class="pl-s">["true" or "false" to indicate whether to provide a download URL]</span>
<span class="pl-ent">google_analytics</span>: <span class="pl-s">[Your Google Analytics tracking ID]</span></pre></div>
<h3><a id="user-content-stylesheet" class="anchor" href="#stylesheet" aria-hidden="true"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Stylesheet</h3>
<p>If you'd like to add your own custom styles:</p>
<ol>
<li>Create a file called <code>/assets/css/style.scss</code> in your site</li>
<li>Add the following content to the top of the file, exactly as shown:
<div class="highlight highlight-source-scss"><pre>---
---

<span class="pl-k">@import</span> "<span class="pl-v">{{</span> <span class="pl-v">site.theme</span> <span class="pl-v">}}</span>";</pre></div>
</li>
<li>Add any custom CSS (or Sass, including imports) you'd like immediately after the <code>@import</code> line</li>
</ol>
<h3><a id="user-content-layouts" class="anchor" href="#layouts" aria-hidden="true"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Layouts</h3>
<p>If you'd like to change the theme's HTML layout:</p>
<ol>
<li><a href="https://github.com/pages-themes/minimal/blob/master/_layouts/default.html">Copy the original template</a> from the theme's repository<br>(<em>Pro-tip: click "raw" to make copying easier</em>)</li>
<li>Create a file called <code>/_layouts/default.html</code> in your site</li>
<li>Paste the default layout content copied in the first step</li>
<li>Customize the layout as you'd like</li>
</ol>
<h2><a id="user-content-roadmap" class="anchor" href="#roadmap" aria-hidden="true"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Roadmap</h2>
<p>See the <a href="https://github.com/pages-themes/minimal/issues">open issues</a> for a list of proposed features (and known issues).</p>
<h2><a id="user-content-project-philosophy" class="anchor" href="#project-philosophy" aria-hidden="true"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Project philosophy</h2>
<p>The Minimal theme is intended to make it quick and easy for GitHub Pages users to create their first (or 100th) website. The theme should meet the vast majority of users' needs out of the box, erring on the side of simplicity rather than flexibility, and provide users the opportunity to opt-in to additional complexity if they have specific needs or wish to further customize their experience (such as adding custom CSS or modifying the default layout). It should also look great, but that goes without saying.</p>
<h2><a id="user-content-contributing" class="anchor" href="#contributing" aria-hidden="true"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Contributing</h2>
<p>Interested in contributing to Minimal? We'd love your help. Minimal is an open source project, built one contribution at a time by users like you. See <a href="/pages-themes/minimal/blob/master/CONTRIBUTING.md">the CONTRIBUTING file</a> for instructions on how to contribute.</p>
<h3><a id="user-content-previewing-the-theme-locally" class="anchor" href="#previewing-the-theme-locally" aria-hidden="true"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Previewing the theme locally</h3>
<p>If you'd like to preview the theme locally (for example, in the process of proposing a change):</p>
<ol>
<li>Clone down the theme's repository (<code>git clone https://github.com/pages-themes/minimal</code>)</li>
<li><code>cd</code> into the theme's directory</li>
<li>Run <code>script/bootstrap</code> to install the necessary dependencies</li>
<li>Run <code>bundle exec jekyll serve</code> to start the preview server</li>
<li>Visit <a href="http://localhost:4000"><code>localhost:4000</code></a> in your browser to preview the theme</li>
</ol>
<h3><a id="user-content-running-tests" class="anchor" href="#running-tests" aria-hidden="true"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Running tests</h3>
<p>The theme contains a minimal test suite, to ensure a site with the theme would build successfully. To run the tests, simply run <code>script/cibuild</code>. You'll need to run <code>script/bootstrap</code> one before the test script will work.</p>
